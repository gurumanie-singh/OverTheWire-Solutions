Level 2: Read File with Spaces

← Back to Guide

Level Goal

The password for the next level is stored in a file called spaces in this filename located in your home directory.

Step-by-Step Solution

1. Exit Level 1 and SSH into Level 2

After retrieving the Level 2 password from cat ./-, exit the current SSH session: 

exit

Then connect as bandit2. For example, if the Level 2 password was 9c9f5072485db6fdfb7cbcb620f9106a, run: 

ssh bandit2@bandit.labs.overthewire.org -p 2220

Enter the Level 2 password when prompted.

2. List Files in Your Home Directory

Once logged in as bandit2, list files: 

ls

You will see a file named spaces in this filename. Because the name contains spaces, cat spaces in this filename will treat each word as a separate argument and fail.

3. Read the File by Escaping Spaces or Using Quotes

To correctly reference the file, either escape each space: 

cat spaces\ in\ this\ filename

Or wrap the entire filename in quotes: 

cat "spaces in this filename"

Either command will display the Level 3 password.

Summary & Key Commands

  • exit : leave the SSH session (Level 1).
  • ssh bandit2@bandit.labs.overthewire.org -p 2220 : connect to Level 2.
  • ls : list files; you'll see spaces in this filename.
  • cat spaces\ in\ this\ filename : read the contents by escaping spaces.
  • cat "spaces in this filename" : alternative: wrap the filename in quotes.
