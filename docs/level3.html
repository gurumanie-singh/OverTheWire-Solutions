Level 3: Spaces in a Filename

← Back to Guide

Level Goal

The password for the next level is stored in a file whose name contains spaces.

Step-by-Step Solution

1. Exit Level 2 and SSH into Level 3

If you’re still logged in as bandit2, type: 

exit

SSH in as bandit3 (using the password retrieved from Level 2, e.g., 9c9f5072485db6fdfb7cbcb620f9106a): 

ssh bandit3@bandit.labs.overthewire.org -p 2220

Enter the Level 2 password when prompted.

2. List Files in Your Home Directory

Once logged in as bandit3, list all files: 

ls

You should see a file named spaces in this filename. Trying cat spaces in this filename would fail because the shell splits on spaces.

3. Read the File by Escaping Spaces or Using Quotes

To reference a filename with spaces, escape the spaces: 

cat spaces\ in\ this\ filename

Or wrap the entire filename in quotes: 

cat "spaces in this filename"

Either command prints the Level 3 password.

4. Summary & Key Commands

  • exit : exit the SSH session (Level 2).
  • ssh bandit3@bandit.labs.overthewire.org -p 2220 : log in to Level 3.
  • ls : list files (you'll see spaces in this filename).
  • cat spaces\ in\ this\ filename : escape spaces to read the file.
  • cat "spaces in this filename" : alternative quoting approach.
