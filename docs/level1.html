Level 1: Read File Named “-”

← Back to Guide

Level Goal

The password for the next level is stored in a file called - located in your home directory.

Step-by-Step Solution

1. Exit Level 0 and SSH into Level 1

After retrieving the Level 1 password from readme, exit the current SSH session: 

exit

Then connect as bandit1. For example, if the Level 1 password was f2f14efb574cd7b3bb7d6a19de566c82, run: 


ssh bandit1@bandit.labs.overthewire.org -p 2220

Enter the Level 1 password when prompted.

2. Change into Your Home Directory and List Files

Once logged in as bandit1, ensure you’re in your home directory (you start there by default). List files: 

ls

You will see a single file named -. Because - is interpreted as an option, trying cat - fails: 

cat -

This returns you to the shell prompt without showing the password, because - is treated as “read from stdin” rather than a filename.

3. Read the File by Specifying Its Path

To force cat to treat - as a filename, prefix it with ./: 

cat ./-

This will display the Level 2 password.

Summary & Key Commands

  • exit : leave the current SSH session (Level 0).
  • ssh bandit1@bandit.labs.overthewire.org -p 2220 : connect to Level 1 using the Level 1 password obtained from readme.
  • ls : list files in your home directory; you’ll see a file named -.
  • cat ./- : read the contents of the file named “-” to retrieve the Level 2 password.
